ODC Data Dictionary Quick Guidelines

Download the Data Dictionary template here .

The data dictionary provides critical information for the interpretability and reusability of the dataset. When you submit a data dictionary, we will verify that:

  • The Data Dictionary includes the columns: VariableName, Title, Unit_of_Measure, Description, and Comments.
  • Every Variable (i.e. column) in the dataset is listed under the VariableName column of the data dictionary.
  • The Data dictionary is a “.csv” formatted file.

When a dataset is submitted for DOI and publication, we will require that a data dictionary is included with the submission. The submitted dictionary must have all 9 required columns filled appropriately and will be validated as part of the submission and publication process.

    1. If you are not submitting the dataset for publication yet, you only need to provide:
    1. VariableName: Column containing the Variable (i.e. Column) names of the dataset
    2. Title: Column containing full names of the Variable. E.g. defining acronyms, etc.
    3. Unit_of_Measure: Column containing units for the Variable if applicable
    4. Description: Column containing additional definitions and descriptions of the variable. The description should explain what the variables represent in enough detail such that a reader can understand the content of the variable.
    5. Comments: Column for additional notes such as reasons for subject exclusion, special values, etc.
    2. If the dataset is being submitted for DOI and publication, you must also provide:
    1. PermittedValues: A list of all possible values or a range of values for quantitative variables.
    2. DataType: Whether the Variable (i.e. column) contains Numeric, Categorical, Ordinal, etc data types.
    3. MaximumValue: Maximum possible value (if applicable)
    4. MinimumValue: Minimum possible value (if applicable)
